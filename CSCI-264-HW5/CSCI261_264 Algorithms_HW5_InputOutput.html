Problem 2

Input specification: the first line contains n and m (the number of vertices and edges of the graph), separated by space. The vertices are labeled with numbers 1, 2, ..., n. Then m lines follow. The i-th of these lines contains two numbers representing the end-points of the i-th edge, separated by space. You may assume that n is at most 10000.

Output specification: output the smallest number of edges that need to be added to make the graph connected.

Sample input: input-1.1, input-1.2, input-1.3, input-1.4, input-1.5, input-1.6, input-1.7, input-1.8, input-1.9
Sample output: answer-1.1, answer-1.2, answer-1.3, answer-1.4, answer-1.5, answer-1.6, answer-1.7, answer-1.8, answer-1.9


How to submit: Name your file ConnectGraph.java or ConnectGraph.cpp


Problem 3

Input specification: the first line contains n, the number of courses. The courses are labeled with numbers 1, 2,..., n. Then n lines follow. The i-th line lists the prerequisites in Pi, separated by spaces, finished with number 0. You may assume that n is at most 100000 and every course has at most 100 prerequisites.

Output specification: the first line contains the number of courses in the longest prerequisite chain.

Sample input: input-1.1, input-1.2, input-1.3, input-1.4, input-1.5
Sample output: answer-1.1, answer-1.2, answer-1.3, answer-1.4, answer-1.5
Explanation for the first output: The longest prerequisite chain is: 2->3->1->4->6, containing 5 courses.


How to submit: Name your file Prerequisites.java or Prerequisites.cpp


Problem 4

Input specification: The first line contains the dimensions a and b. Then a lines follow, each contains a string of length b of symbols ".", "x", "1", and "2", where "1" stands for the initial location of thing One, "2" for thing Two, "." is an empty space that the things can move to, and "x" is a wall. Once thing One or Two moves from its initial location, the location becomes a ".". You may assume that a,b are at most 100.

Output specification: Output the smallest number of moves thing One and thing Two need to get out of the house; or the string STUCK if they cannot get out.

Sample input: input-3.1, input-3.2, input-3.3, input-3.4, input-3.5, input-3.6, input-3.7
Sample output: answer-3.1, answer-3.2, answer-3.3, answer-3.4, answer-3.5, answer-3.6, answer-3.7


How to submit: Name your file DoubleTrouble.java or DoubleTrouble.cpp


Problem 5

Input specification: the first line contains n and m (the number of vertices and edges of the graph), separated by a space. The vertices are labeled with numbers 1, 2, ..., n. The second line contains s and t, separated by a space. Then m lines follow. The i-th of these lines contains two numbers representing the end-points of the i-th edge, separated by a space. You may assume that n is at most 1000.

Output specification: output the number of shortest paths from s to t. You may assume that the number fits in int.

Sample input: input-1.1, input-1.2, input-1.3, input-1.4, input-1.5, input-1.6, input-1.7
Sample output: answer-1.1, answer-1.2, answer-1.3, answer-1.4, answer-1.5, answer-1.6, answer-1.7


How to submit: Name your file NumPaths.java or NumPaths.cpp
